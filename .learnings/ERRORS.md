## 2026-06-08 - better-sqlite3 require from workspace root failed
- Context: while debugging FreeChat DB state, running `node - <<... require('better-sqlite3')` from repo root failed with MODULE_NOT_FOUND.
- Cause: dependency is installed under `packages/server`; use `pnpm --filter @freechat/server exec node ...`, import server db module, or run from `packages/server`.
- Fix pattern: for package-local deps in pnpm workspace, execute within the package context.

## [ERR-20260612-CLI-ESM-WRAPPER] FreeChat Agent CLI executed as ESM while using require

**Logged**: 2026-06-12T22:06:00+08:00
**Status**: fixed
**Impact**: Agent `./freechat file write-local` could not publish private `res/` files to project files. Agents reported “CLI bug / file saved” while files only existed in private workspace.

**Symptom**:
```text
ReferenceError: require is not defined in ES module scope
at .../agents/<agentId>/freechat:2 const fs = require('fs')
```

**Root cause**: repo package uses `"type":"module"`; generated extensionless `./freechat` was treated as ESM, but template used CommonJS `require`.

**Fix**: generate `./freechat` as a bash wrapper and put real CommonJS CLI in `.freechat/freechat.cjs`. Refreshed current workspace wrapper and published previously private novel chapter files to project file area.

**Verification**: `./freechat tool list` works in Agent workspace; `file write-local` published `res/星源纪/正文/第一卷/第一章.md` and `第二章.md`; `pnpm check` passed.
